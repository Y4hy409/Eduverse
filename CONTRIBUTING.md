# Contributing

Thank you for considering a contribution to EduVerse.

## Development workflow

1. Fork the repository and create a branch from `main`.
2. Serve the project with a local static web server.
3. Make a focused change and test each affected role.
4. Open a pull request with a clear summary and testing notes.

For example, start a local server with Python:

```bash
python -m http.server 8000
```

Then visit `http://localhost:8000`.

## Pull requests

- Keep each pull request focused on one improvement.
- Preserve the existing visual language and responsive behavior.
- Test student, teacher, and administrator flows when relevant.
- Include screenshots for visible interface changes.
- Do not commit secrets, generated files, or editor configuration.

## Reporting issues

Include the browser, viewport size, affected role, steps to reproduce, expected
behavior, and actual behavior. Screenshots or console errors are helpful.
